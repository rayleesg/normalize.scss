normalize.scss

Documentation

Normalize.scss permet de configurer simplement et rapidement normalise.css. Choisissez les règles, ajoutez ou supprimez les commentaires explicatifs et le support d'IE7.

Comment utiliser normalize.scss

  • Téléchargez normalize.scss, puis copiez les fichiers à l'endroit où vous placez vos fichiers CSS.
  • Lancez la ligne de commande suivante pour que Sass génère votre fichier css : 
    sass --watch normalize.scss:normalize.css
  • Ouvrez le fichier normalize.scss dans votre éditeur de texte et modifiez les variables de configuration selon vos besoins.

Configurer normalize.scss

La configuration se passe dans le fichier normalize.scss. Elle est diviser en 2 parties principales : options et parts. 

$normalize-settings: (

	"options": (
		...
	),

	"parts": (
		...
	)

);

Options

"options": (
	"explanatory comments":true,
	"legacy support for ie7":false
),
explanatory comments
  • true : ajoute les commentaires explicatifs de normalize.css
  • false : supprime les commentaires explicatifs de normalize.css
legacy support for ie7
  • true : ajoute le support pour IE7
  • false : supprime le support pour IE7

Pour plus d'informations sur le support d'IE 7, reportez vous à la partie "Le support d'IE7".

Le support d'IE 7

Normalize.css ne prend plus en charge le support des anciens navigateurs tels que IE6, IE7 ou encore safari 4. La branche qui supportait ces navigateurs n'est plus développée (Normalize.css v1).

Dans un cadre professionnel, on peut se permettre de dire non à IE6, mais IE7 reste encore présent pour certaines entreprises. C'est pourquoi normalize.scss propose le support pour ce navigateur.

Je maintiens une version de normalize.css supportant IE7 à jour, l’option de support IE7 de normalize.scss est basée dessus.

Legacy support for IE 7 : true;

L'activation du support d'IE 7 ajoute son lot de règles et de hacks :

Base
  • Correction d'un bug de redimensionnement de la typo lorsque font-size est définie sur le body avec l'unité de valeur em.
html {
    font-family: sans-serif; /* 1 */
    font-size: 100%; /* 2 */
    -ms-text-size-adjust: 100%; /* 3 */
    -webkit-text-size-adjust: 100%; /* 3 */
}
HTML5 display definitions
  • Hack haslayout pour les déclarations de display en inline-block.
audio,
canvas,
progress,
video {
    display: inline-block; /* 1 */
    *display: inline; /* 2 */
    *zoom: 1; /* 2 */
    vertical-align: baseline; /* 3 */
}
Text-level semantics
  • La taille de la typo et les marges externes des headers sont différents sur IE 7.
h2 {
    font-size: 1.5em;
    margin: 0.83em 0;
}

h3 {
    font-size: 1.17em;
    margin: 1em 0;
}

h4 {
    font-size: 1em;
    margin: 1.33em 0;
}

h5 {
    font-size: 0.83em;
    margin: 1.67em 0;
}

h6 {
    font-size: 0.67em;
    margin: 2.33em 0;
}
  • Les marges externes des p sont différentes sur IE 7.
p {
    margin: 1em 0;
}
  • l'attribut CSS quotes n'est pas reconnu sur IE 7
q {
    quotes: none;
}
Lists

La partie "Lists" a été enlevée de normalize.css en même temps que l'arrêt du support des anciens navigateurs.

  • Les marges externes et internes sont différentes sur IE 7.
  • Les puces des listes sont mal gérées.
/* Lists
   ======================================================= */

/**
* Address margins set differently in IE 7.
*/

dl,
ol,
ul {
    margin: 1em 0;
}

dd {
    margin: 0 0 0 40px;
}

/**
* Address paddings set differently in IE 7.
*/

ol,
ul {
    padding: 0 0 0 40px;
}

/**
* Correct list images handled incorrectly in IE 7.
*/

nav ul,
nav ol {
    list-style: none;
    list-style-image: none;
}
Embedded content
  • Utilisation de l'attribut -ms-interpolation-mode pour améliorer la qualité des images lorsque l'image est redimensionnée.
img {
    border: 0; /* 1 */
    -ms-interpolation-mode: bicubic; /* 2 */
}
Grouping content
  • L'élément hr n'a pas le bon display sur IE 7.
hr {
	display: block;
	-moz-box-sizing: content-box;
	box-sizing: content-box;
	height: 0;
}
  • Les marges externes de pre sont différentes sur IE 7.
pre {
    margin: 1em 0; /* 1 */
    overflow: auto; /* 2 */
}
Forms
  • Les marges externes de form sont interprétées bizarrement sur IE 7.
form {
    margin: 0;
}
  • Correction des espaces internes de certaines balises sur IE 7.
button,
html input[type="button"], /* 1 */
input[type="reset"],
input[type="submit"] {
    -webkit-appearance: button; /* 2 */
    cursor: pointer; /* 3 */
    *overflow: visible; /* 4 */
}
  • Correction d'un bug de tailles sur les champs de type radio et checkbox.
input[type="checkbox"],
input[type="radio"] {
    box-sizing: border-box; /* 1 */
    padding: 0; /* 2 */
    *height: 13px; /* 3 */
    *width: 13px; /* 3 */
}
  • Correction d'un bug d'alignement sur legend.
legend {
    border: 0; /* 1 */
    padding: 0; /* 2 */
    *margin-left: -7px; /* 3 */
}
