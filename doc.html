How to use normalize.scss

  • Download normalize.scss, and copy the files to where you put your CSS files.
  • Run the following command line for Sass generates your css file: 
    sass --watch normalize.scss:normalize.css
  • Open normalize.scss in your text editor and change the configuration variables as needed.

Configure normalize.scss

The configuration happens in the normalize.scss file. It is divided into two main parts: options and parts. 

$normalize-settings: (

	"options": (
		...
	),

	"parts": (
		...
	)

);

Options

"options": (
	"explanatory comments":true,
	"legacy support for ie7":false
),
explanatory comments
  • true: add explanatory comments to normalize.css
  • false: remove explanatory comments to normalize.css
legacy support for ie7
  • true: add support for IE7
  • false: remove support for IE7

For more information on IE ​​7 support , refer to the part "IE ​​7 support".

Parts: base

"base": (
	"html":true,
	"reset margin":"body"
),
html
  • true: add the following code
  • false: remove the following code
/**
 * 1. Set default font family to sans-serif.
 * 2. Prevent iOS text size adjust after orientation change, without disabling
 * user zoom.
 */

html {
  font-family: sans-serif; /* 1 */
  -ms-text-size-adjust: 100%; /* 2 */
  -webkit-text-size-adjust: 100%; /* 2 */
}
reset margin
  • "list of tags, in quotes, separated by commas": add the following code
  • false: remove the following code
/**
 * Remove default margin.
 */

body {
  margin: 0;
}

IE 7 support

Normalize.css no longer supports older browsers such as IE 6, IE 7, or Safari 4. The branch that supported these browsers is no longer developed (Normalize.css v1).

In the workplace, you can afford to say no to IE 6, but IE 7 still present for some companies. That is why normalize.scss offers support for this browser.

I maintain an updated version of normalize.css supporting IE7, the option of IE7 support normalize.scss is based on.

Legacy support for IE 7 : true;

Activation of IE 7 support adds a lot of rules and hacks:

Base
  • Correct text resizing oddly in IE 7 when body font-size is set using em units.
html {
    font-family: sans-serif; /* 1 */
    font-size: 100%; /* 2 */
    -ms-text-size-adjust: 100%; /* 3 */
    -webkit-text-size-adjust: 100%; /* 3 */
}
HTML5 display definitions
  • Haslayout hack for display in inline-block.
audio,
canvas,
progress,
video {
    display: inline-block; /* 1 */
    *display: inline; /* 2 */
    *zoom: 1; /* 2 */
    vertical-align: baseline; /* 3 */
}
Text-level semantics
  • For the headers, address font sizes and margins set differently in IE 7.
h2 {
    font-size: 1.5em;
    margin: 0.83em 0;
}

h3 {
    font-size: 1.17em;
    margin: 1em 0;
}

h4 {
    font-size: 1em;
    margin: 1.33em 0;
}

h5 {
    font-size: 0.83em;
    margin: 1.67em 0;
}

h6 {
    font-size: 0.67em;
    margin: 2.33em 0;
}
  • For p, address margins set differently in IE 7.
p {
    margin: 1em 0;
}
  • For q
  • , Address CSS quotes not supported in IE 7.
q {
    quotes: none;
}
Lists

"Lists" part has been removed from normalize.css simultaneously stop supporting older browsers.

  • Address margins and paddings set differently in IE 7.
  • Correct list images handled incorrectly in IE 7.
/* Lists
   ======================================================= */

/**
* Address margins set differently in IE 7.
*/

dl,
ol,
ul {
    margin: 1em 0;
}

dd {
    margin: 0 0 0 40px;
}

/**
* Address paddings set differently in IE 7.
*/

ol,
ul {
    padding: 0 0 0 40px;
}

/**
* Correct list images handled incorrectly in IE 7.
*/

nav ul,
nav ol {
    list-style: none;
    list-style-image: none;
}
Embedded content
  • Improve image quality when scaled in IE 7.
img {
    border: 0; /* 1 */
    -ms-interpolation-mode: bicubic; /* 2 */
}
Grouping content
  • Correct display set differently for hr in IE 7.
hr {
	display: block;
	-moz-box-sizing: content-box;
	box-sizing: content-box;
	height: 0;
}
  • For pre, address margins set differently in IE 7.
pre {
    margin: 1em 0; /* 1 */
    overflow: auto; /* 2 */
}
Forms
  • For form, correct margin displayed oddly in IE 7.
form {
    margin: 0;
}
  • Remove inner spacing in IE 7 without affecting normal text inputs.
button,
html input[type="button"], /* 1 */
input[type="reset"],
input[type="submit"] {
    -webkit-appearance: button; /* 2 */
    cursor: pointer; /* 3 */
    *overflow: visible; /* 4 */
}
  • Remove excess padding in IE 7.
input[type="checkbox"],
input[type="radio"] {
    box-sizing: border-box; /* 1 */
    padding: 0; /* 2 */
    *height: 13px; /* 3 */
    *width: 13px; /* 3 */
}
  • Correct alignment displayed oddly in IE 7 for legend
legend {
    border: 0; /* 1 */
    padding: 0; /* 2 */
    *margin-left: -7px; /* 3 */
}
